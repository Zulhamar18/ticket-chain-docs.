Pendahuluan tentang Pembayaran A2U (App-to-User)

Sementara Pi Frontend Javascript SDK mendukung pembuatan pembayaran U2A (User-to-App), aplikasi Anda mungkin memerlukan kemampuan untuk membuat pembayaran A2U (App-to-User). Fitur ini saat ini hanya tersedia di Testnet.

Pembayaran App-to-User

Desain saat ini untuk pembayaran A2U melibatkan interaksi dengan Pi Blockchain dan Pi Backend. Pi Blockchain adalah satu-satunya sumber kebenaran untuk pertukaran Pi. Pi Backend digunakan untuk meningkatkan pengalaman pengguna, seperti memberikan memo yang dapat dimengerti pada dompet mereka, dan tautan balik ke aplikasi Anda.

Manfaat Utama:

  • Privasi Pengguna yang Lebih Baik: API ini memungkinkan akses aman ke alamat dompet pengguna selama mereka telah menyetujui untuk berbagi dengan aplikasi Anda.
  • Akurasi Dompet Pengguna: API mengembalikan dompet pengguna saat ini jika perubahan dompet diperlukan, untuk menghindari aplikasi mengirim Pi ke alamat dompet yang sudah tidak digunakan.
  • Pengalaman yang Lebih Baik untuk Pengguna dan Pengembang Aplikasi: Menginformasikan pembayaran ke Pi Backend sebelum dan sesudah transaksi membantu menghindari pembayaran ganda atau bug lainnya.
  • Integrasi yang Lebih Baik dengan Ekosistem Pi: Menggunakan API A2U memungkinkan pembayaran ditampilkan di dompet pengguna sebagai berasal dari aplikasi Anda, bukan sebagai transaksi acak dari alamat tidak dikenal.
